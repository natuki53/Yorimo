# Yorimo デザイン

このディレクトリでは、Yorimo のプロダクトデザイン資産を管理します。デザインシステム、Penpot 取り込み用ファイル、確認用プロトタイプ、書き出し済みアイコンを役割別に配置しています。

## 構成

| パス | 用途 |
| --- | --- |
| `system/README.md` | グローバルなデザインシステムとアプリ全体のルール。 |
| `system/tokens.css` | 今後のフロントエンド実装で使うデザイントークンと共通コンポーネントの土台。 |
| `penpot/yorimo-mobile-app.svg` | Penpot に取り込むメインファイル。MVP の全モバイル画面を含みます。 |
| `penpot/yorimo-exploration.svg` | 初期検討用の4画面ボード。 |
| `prototypes/mobile-app.html` | すばやく確認するための静的ブラウザプロトタイプ。 |
| `assets/icons/app-icon.png` | 生成した透過アプリアイコン。 |
| `assets/yorimo-presentation-qr.png` | 発表スライド用QRコード（`https://yorimo.mu-natuki.com/`）。 |
| `assets/yorimo-presentation-qr.svg` | 拡大しても劣化しない発表スライド用QRコード。 |
| `assets/icons/yorimo-icon-set.svg` | Penpot コンポーネント用の生成 SVG アイコンセット。 |

## Penpot への取り込み

1. Penpot を開きます。
2. Yorimo 用のファイルを新規作成、または既存ファイルを開きます。
3. `penpot/yorimo-mobile-app.svg` をキャンバスへドラッグするか、Penpot の取り込み機能を使います。
4. 個別のフレームや要素を編集したい場合は、取り込んだボードをグループ解除します。
5. 繰り返し使う UI は Penpot コンポーネントとして再構成します。対象は、マップヘッダー、推薦カード、ボトムナビ、チップ、アイコンボタン、テキスト入力、生成アイコンです。

## 確認用プロトタイプ

`prototypes/mobile-app.html` をブラウザで開くと、静的な画面確認ができます。共通トークンは `system/tokens.css` から読み込みます。

## デザイン意図

Yorimo はランディングページではなく、モバイルファーストの実用アプリとして設計します。最初の画面では、現在のルート、使える時間、予算、気分フィルター、最適な寄り道候補を表示するボトムシートを含め、実際に使えるマップ体験を見せます。

Instagram と X の雰囲気を参考にしますが、各サービスのロゴや固有のプロダクト表現はコピーしません。

- Instagram: グラデーションアクセント、ソーシャル感、アクティブリング、投稿時の温度感。
- X: モノクロ中心の情報整理、コンパクトな操作、強い文字階層、アイコン優先のアクション。
- Google Maps: 見慣れた明るいマップベース、カスタムルート、公園、水域、スポット表示。
- リール / ストーリー: Yorimo のショート動画と一時投稿として含め、スポット詳細や安全導線につなげます。

視覚バランスの基本方針は次の通りです。

- 落ち着いたマップ実用性: Google Maps に近い淡い地面、白い道路、青いルート。
- ソーシャル感: トップ推薦、投稿、アクティブ状態に Instagram 風のグラデーションを使用。
- 情報の読みやすさ: X 風の白黒カード、コンパクトなメタ情報、シンプルなアイコンボタン。
- 信頼と完了状態: 保存済み、訪問済みには Google 系のグリーンを使用。
- 実用上の注意: 時間、予算、注意喚起にはアンバー系を使用。

## 含まれる画面

メインボードには14個のモバイル画面を含みます。

1. ログイン
2. 登録 / 初期プロフィール設定
3. ホームマップ
4. おすすめ一覧
5. スポット詳細
6. 投稿作成: 写真、リール / ショート動画、ストーリー、レビュー
7. フィード
8. 保存スポット
9. マイルート
10. プロフィール
11. コンテンツ通報
12. ユーザーブロック
13. リール / ショート動画
14. ストーリー閲覧

## UI に反映しているバックエンド仕様

- `POST /api/recommendations` の入力項目は、現在地、ルート、利用可能時間、予算、気分、興味タグとして画面上のフィルターに反映します。
- スコアの内訳は、隠れたランキングロジックではなく理由チップとして表示します。
- `followers` の公開範囲はバックエンド enum には存在しますが、フォロー関係は未実装のため「準備中」として表示します。
- ユーザー生成コンテンツの画面では、通報とブロックの導線を見える位置に置きます。
