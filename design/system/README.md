# Yorimo Penpot デザインシステム

これは Penpot 向けリデザインで使うデザインシステムの管理用スナップショットです。`ui-ux-pro-max` で生成した方向性をベースに、Yorimo のプロダクト要件、バックエンド仕様、Instagram / X を参考にした見た目の方向性へ調整しています。

## プロダクト原則

- ユーザーが「今どこに寄り道できるか」を30秒以内に判断できるようにする。
- マップは常に見えて操作できる状態に保つ。カードはマップを置き換えるものではなく、判断理由を補足するものにする。
- ルートとの相性、使える時間、予算、気分は推薦入力そのものなので、画面上で見えるようにする。
- 安全性、公開範囲、通報、ブロックは補助機能ではなく、主要な UI 要件として扱う。
- ソーシャルな場面には Instagram 風のグラデーションの勢いを使い、情報量の多い画面では X 風のモノクロで抑制する。
- 位置判断が直感的になるよう、マップはカスタマイズした Google Maps に近い見え方を保つ。

## ビジュアル方針

**スタイル:** Instagram / X 風のソーシャルマップユーティリティ  
**ムード:** ソーシャル、即時性、クリーン、信頼感、読みやすさ  
**構造:** モバイルファーストのマップキャンバス、ボトムシート、コンパクトな推薦カード、明確なフィルターチップ、安全な投稿フロー。

アプリ本体では、ランディングページのような見せ方を避けます。Yorimo は実用アプリなので、最初の表示領域ではマップと推薦が実際に使える状態で見えている必要があります。

## 参考デザインの取り込み方

- **Instagram 参考:** グラデーションアクセント、ストーリー風のアクティブリング、勢いのある推薦スコア、温かみのある投稿画面。
- **X 参考:** 白黒を軸にした情報階層、コンパクトなタイポグラフィ、強い区切り線、アイコン優先のシンプルなアクション、フィード型カード。
- **Google Maps 参考:** 明るいマップベース、淡い地面、白い道路、青いルート線、緑の公園、淡い水色の水域、見慣れた浮遊マップコントロール。
- Instagram、X、Google のブランドマークはコピーしません。操作感と配色ロジックだけを参考にします。

## コンポーネント

### アイコンシステム

- カスタム SVG ラインアイコンは `../assets/icons/yorimo-icon-set.svg` で管理します。
- ストロークは丸い線端を使い、24px アイコン換算で 2.25-2.5px 程度にします。通常状態は X 風のモノクロを基本にします。
- アクティブなソーシャル状態には、Instagram 風グラデーションまたはピンクアクセントを使います。
- Like / ハートアイコンは、縦の中心線に対して数学的に左右対称にします。
- 通報とブロックのアイコンは、同じ危険色 `#E1306C` と背景 `#FDE7F0` を使います。
- マップ系ユーティリティアイコンには Google Maps 風の青 `#1A73E8` を使います。
- プロダクト UI では、絵文字、ブランドロゴ、混在したアイコンパックを使いません。
- アプリアイコンは `../assets/icons/app-icon.png` で管理し、Instagram / X / Google のロゴをコピーせず、Yorimo 独自のルート形状マークを使います。

### マップヘッダー

- 上部に浮遊検索フィールドを置き、ルート文脈と現在のモードを表示します。
- 高さは 56-64px に抑えます。
- ルートアイコン、検索アイコン、フィルターボタンを1つずつ使います。
- 入力フォーカスリングはマップルートの青 `#1A73E8` を使い、3px のアウトラインにします。

### 推薦シート

- 見えるドラッグハンドル付きのボトムシートにします。
- 1行目には現在の意図を要約します。例: `45分 / 1500円 / 甘いもの`。
- カードにはスコア、時間、予算、理由チップ、主要アクションを表示します。
- シートを半分まで展開しても、現在地マーカーを隠さないようにします。

### スポットカード

- 必須情報: 名前、カテゴリ、ルートとの相性、滞在時間、予算、推薦理由を1つ。
- 任意情報: 写真、評価、タグ、直近の公開レビューのプレビュー。
- アクション: 保存、スキップ、訪問済み。アイコンボタンは 44px のタッチターゲットにします。

### 投稿作成

- バックエンドの投稿タイプ `photo`、`short_video`、`story`、`review` をすべて扱います。
- ユーザー向けの呼び方では、`short_video` を「リール」として扱います。
- 最初はコンパクトなソーシャル投稿フォームにし、投稿タイプを選んだあとでメディア種別ごとの入力へ分岐します。
- 動画画面には、ミュート、一時停止、保存、通報、スポット詳細への明確なアクションを用意します。

### 安全面

- 投稿詳細やユーザー生成コンテンツの画面には、必ず通報 / ブロック導線を置きます。
- 公開範囲セレクターでは現在の挙動を説明します。`followers` は完全実装されていないため、フォロー機能が実装されるまでは「準備中」と表示します。

## メインアプリの上書きルール

- 標準モバイルフレームは 390px x 844px。
- ヘッダーは左右16px内側、56px高、マップ上に浮かせます。
- ボトムナビは固定表示、5項目、64px高、ラベル付きにします。
- フル幅の主要ボタンは、390px モバイルフレーム内で `x=34`、`width=322` を基準にします。
- 横並びのアクションボタンは 44px x 44px のタッチターゲットにし、8px以上の間隔を取り、左右の余白を揃えます。
- 推薦シートは下固定、上角20px、8pxのドラッグハンドルを使います。
- ルート線は Google Maps 風の青 `#1A73E8`、4px 線にします。
- 現在地マーカーは青塗り、白リング、控えめな精度ハローを使います。
- 推薦スポットのマーカーは、トップ候補には Instagram 風グラデーションリング、通常候補には白黒、保存済みにはグリーンを使います。
- マップ上の主要 CTA は Instagram 風グラデーションにします。補助アクションは白地に X 風の黒テキストを使います。
- 通報 / ブロックなどの破壊的 CTA は黒ではなく、単色の `#E1306C` を使います。
- マップベースはカスタマイズした Google Maps に近づけます。淡い地面、白い道路、青い水域、緑の公園、簡略化したスポットラベルを使います。

## アクセシビリティ

- 通常テキストのコントラスト比は 4.5:1 を満たします。
- タッチターゲットは最低 44px x 44px にします。
- ボトムナビにはアイコンだけでなくテキストラベルも付けます。
- ローディング状態ではスケルトンまたは進捗表示を使い、空のマップや空のリストを放置しません。
- トランジションは 150-300ms にし、`prefers-reduced-motion` を尊重します。

## 避けること

- UI アイコンに絵文字を使わない。SVG アイコンを使います。
- 装飾的なグラデーションを主役の視覚要素にしない。
- 大きすぎるカードでマップコントロールを隠さない。
- 理由、予算、安全説明に小さすぎる薄いグレー文字を使わない。
- フォロー関係が実装されるまで、`followers` を実際の公開範囲として扱わない。
- 音声を自動再生しない。リールとストーリーはミュート状態で開始し、操作ボタンを見えるようにします。
- MVP では、リール、動画フィード、ストーリー中心の導線をアプリ全体の主軸にしない。
